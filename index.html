📡 Modelo Conceptual: InpeCable

Gestión de Telecomunicaciones y Facturación

1. El Caso de Negocio

InpeCable requiere organizar de manera estructurada su información operativa. Este modelo de base de datos facilita el control del ciclo de vida de las suscripciones, el registro de facturación y el seguimiento de incidencias técnicas.

2. Entidades Principales

👤 Cliente

Registra datos personales, contacto y dirección principal del titular.

🌐 Plan Servicio

Catálogo centralizado de planes de internet y cable con sus precios.

📄 Contrato

Vincula al cliente con un plan, controlando su estado de suscripción.

💳 Factura

Control mensual de cobros, fechas de vencimiento y morosidad.

🛠️ Ticket Soporte

Gestión y estado de las incidencias técnicas reportadas.

3. Reglas y Relaciones Clave

  • Un Cliente puede firmar múltiples Contratos (1:N).
  • Un Plan de Servicio puede ser asignado a muchos Contratos (1:N).
  • Un Contrato genera múltiples Facturas a lo largo del tiempo (1:N).
  • Un Cliente puede reportar múltiples Tickets de Soporte (1:N).

4. Valor del Modelo de Datos

El diseño permite realizar consultas ágiles y vitales para la gerencia, tales como identificar clientes con facturas vencidas (estado "Pendiente"), calcular ingresos mensuales o conocer los planes más rentables.
